Privacy

agentOS is a local desktop tool. It reads your AI-assistant configuration so it can show and analyze it, and it runs that analysis on your own machine. By default, your configuration, file contents, and prompts stay on your device and are never sent to us or to any third party. Two explicit, opt-in actions can send content off your device, and only to a destination you configure: LLM connectivity, which sends one or more resources you choose to an AI endpoint (for example, a pair of conflicting resources when you ask the model to resolve an overlap; see "LLM connectivity"), and skill sync via the agentos command-line tool, which can copy skill files into a Git repository and push them to your Git remote (see "Command-line tool and skill sync"). Neither happens unless you trigger it, and in both cases agentOS itself never receives your content.

Last updated: 2026-06-14.

What agentOS reads (locally)

  • Your global Claude config under ~/.claude (skills, agents, commands, rules, CLAUDE.md, settings, MCP servers, hooks).
  • When you open a project: that project's .claude, .cursor, .github/copilot-instructions.md, and CLAUDE.md.
  • Your Claude Code session transcripts under ~/.claude/projects, in two cases: (1) when you open a session in the Telemetry view, to show what each tool call did; and (2) if you turn on "Prompt evidence" (off by default, see below), to find real past prompts that would trigger two overlapping skills, agents, or commands.
  • When you click a file or folder path anywhere in the app (the "peek" preview), the contents of that file, or the list of names inside that folder, so it can be shown in the popover. Only paths inside your home folder or a project you have opened can be previewed, and credential files (.env, keys, tokens) stay hidden unless you opt in under Settings. The preview is read and rendered locally and is never uploaded.

These are read to display and analyze them. agentOS only writes to a file when you ask it to (toggling a resource on or off, editing a resource's body, or adding a tag).

The agentos command-line tool reads the same local config from your terminal instead of the window; it reads the same files for the same purpose and sends nothing off-device on its own (its one off-device action, sync push, is described under "Command-line tool and skill sync").

Prompt evidence (off by default)

Conflict analysis can show you a real prompt from your own history that would match two resources at once, so you can see the overlap concretely instead of as a percentage. This is off by default. When you turn it on (Settings, "Prompt evidence"), agentOS reads the user prompts from your local transcripts, compares them against your resource descriptions on your machine, and may display a short, truncated prompt next to a conflict. Your prompts are read and compared locally only and are never uploaded. Turn it off at any time to stop reading transcripts for this purpose.

What agentOS stores (locally, on your device)

  • Usage counts and your tags, under ~/.claude/agentos and ~/.agentos.
  • Your preferences, under ~/.agentos/settings.json (including whether "Prompt evidence" is on).
  • Your connectivity settings, under ~/.agentos/connectivity.json (whether it's enabled, provider, model, base URL) — this file never contains your API key.
  • Your API key for LLM connectivity, encrypted by the operating-system keychain (via Electron safeStorage) under ~/.agentos/connectivity.key. It is never written in plaintext and is never sent back to the app's interface — agentOS only knows whether a key is present.
  • Conflict-analysis vectors, and (when "Prompt evidence" is on) the mined prompt text and its vectors, held in memory for the current session only. They are not written to disk in this version.

Network connections

agentOS works offline once set up. The first time you use conflict analysis, it downloads two things from public CDNs, and only then:

  • the language model (all-MiniLM-L6-v2, about 23 MB) from the HuggingFace CDN (huggingface.co, cas-bridge.xethub.hf.co); and
  • the WebAssembly runtime that executes it (onnxruntime-web) from the jsDelivr CDN (cdn.jsdelivr.net).

These CDNs see your IP address and which files you requested. They do not receive your skills, descriptions, prompts, file contents, or usage data.

After both are cached, the analysis runs entirely on your device with no further network calls. None of your content ever leaves your machine. (A future version may bundle the runtime to drop the jsDelivr dependency and allow fully offline first use.)

To clear the cached model, clear the application's cache (it lives in the app's standard cache storage and is re-downloaded on next use).

LLM connectivity (off by default)

agentOS can optimize a skill, agent, or command's description and contents using a language model, and can use the same model to resolve a conflict between two overlapping resources. This is the only in-app (window) feature that sends your content off your device (skill sync, below, is the other off-device action, and runs from the command line), and it is off by default.

  • Nothing is sent until you opt in. You must enable connectivity and store an API key in the Connectivity menu. Even then, content is only sent when you press Optimize on a specific resource, or use a model-driven action in the conflict resolver, and only for the resource(s) involved.
  • What is sent: for Optimize, the chosen resource's description and body, plus the names and descriptions of its same-kind siblings. For conflict resolution, the description and body of the two resources in the pair you are resolving. Nothing else — no other files, no transcripts, no usage data.
  • Where it goes: only to the endpoint you configure (by default https://api.anthropic.com for the Anthropic provider, or an OpenAI-compatible base URL you set). It does not go to agentOS or its authors. Once the request reaches that endpoint, the provider's own privacy terms govern what they do with it.
  • Your API key is held in the operating-system keychain (see "What agentOS stores"), encrypted, never in plaintext, and is sent only to that endpoint as the request credential.
  • Turn it off at any time in the Connectivity menu; you can also clear the stored key there.

Command-line tool and skill sync

agentOS ships an optional command-line tool, agentos, for reading your configuration and for skill sync — copying skills you select from your global config (~/.claude/skills) into a project so they travel to other machines and remote Claude sessions through Git.

  • Nothing leaves your device until you choose to push. agentos sync is a dry run by default (it only prints what it would copy). With --apply it copies the selected skill folders into that project's .claude/skills on your own disk. With --commit it makes a local Git commit. None of these contact any network.
  • What can leave your device: only when you add --push (which asks you to confirm) does it run git push, sending the committed skill files to the Git remote already configured for that repository.
  • Where it goes: only to your Git remote (for example your GitHub/GitLab account or a server you control). It does not go to agentOS or its authors. Once pushed, that remote's own terms govern the data, and anyone with access to that repository can read the skills.
  • What is sent: only the skill files you selected. No other configuration, transcripts, prompts, keys, or usage data.

You choose which skills to sync and whether to push; the tool never pushes on its own.

Telemetry

Local usage logging only. The optional sync setting is off by default and performs no upload in this version.

Dependencies and security

  • New dependencies are checked for active supply-chain compromise (for example the Shai-Hulud npm campaigns) before installation.
  • Dependencies are installed with --ignore-scripts (no install-time code execution) and pinned to exact versions with a committed lockfile.
  • The embedding model runs on WebAssembly in the renderer. No native add-ons are built or used.
  • npm audit advisories are limited to the development-time build toolchain (esbuild / vite), which is not shipped to end users. The runtime dependency tree carries no known critical or high advisories.

Changes to this policy

This document is updated whenever a new data path is added, in the same change that adds it. Reading conversation transcripts (from ~/.claude/projects) to improve conflict analysis is described above under "Prompt evidence". It runs locally, is off by default, and can be turned off at any time. Sending a resource to a language model for optimization is described under "LLM connectivity"; it is off by default and goes only to an endpoint you configure. The other off-device path, skill sync (agentos sync --push), is described under "Command-line tool and skill sync"; it runs only when you invoke it, sends only the skills you select, and goes only to your own Git remote.
