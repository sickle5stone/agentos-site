How it works

Specific by design.

01

Reads from disk

It opens ~/.claude directly. No agent, no daemon, no sync. The files stay the source of truth.

02

Embeds on-device

Each description is embedded with MiniLM in WebAssembly. The overlap math runs in your window, never a server.

03

Writes atomically

Toggling a resource moves the real file through an atomic rename, the same path Claude Code resolves. Only when you act.

What leaves your machine

No account. No telemetry. Nothing leaves your machine by default.

Optimizer (you press it) skill sync --push (asks first)

Exactly two opt-in egress points. Both off by default. Everything else is local.
