Last updated: 2026-06-14.

The basics

  • agentOS is provided "as is", without warranty of any kind. You use it at your own risk.
  • It reads and, only at your explicit direction, edits configuration files that already exist on your machine. You are responsible for the changes you make through it (enabling or disabling resources, editing their contents, adding tags).
  • With your opt-in ("Prompt evidence", off by default), it also reads your local Claude Code transcripts to find example prompts for overlapping resources. This runs locally. See Privacy.
  • With your opt-in ("LLM connectivity", off by default), it can send one or more resources you choose — a single resource to optimize, or a pair of resources when you resolve a conflict — to a language-model endpoint you configure, using an API key you provide. You are responsible for that key, for any usage costs the provider charges, and for complying with that provider's terms. agentOS sends your content only to the endpoint you set, never to us. See Privacy.
  • The optional agentos command-line tool can sync skills you select into a Git repository and, only when you pass --push and confirm, push them to the Git remote you have configured. You are responsible for choosing what to sync, for the remote you push to, and for who can then access that repository. agentOS pushes only the skills you select and only to your remote, never to us. See Privacy.
  • Apart from these opt-in actions, agentOS does not transmit your configuration or content. See Privacy for what touches the network.
